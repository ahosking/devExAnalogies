# DevEx Analogies

A collection of analogies for explaining developer experience and productivity concepts to different audiences.

The goal is practical: these are conversation tools for when someone says "we need more PRs!" or "why can't you just measure lines of code?" Each analogy has been tested in real conversations and refined over time.

## Analogies

| Analogy | Description |
| ------- | ----------- |
| [Cycling: Cadence + Power = Speed](analogies/cycle-time.md) | Why cycle time alone is insufficient. True productivity requires balancing speed (cadence) with impact (power). |
| [Input vs. Output Metrics](analogies/input-output-metrics.md) | The difference between activity (inputs) and outcomes (outputs). Why measuring the wrong thing backfires. |
| [Health and Family Analogies](analogies/health.md) | A collection of health analogies for reframing metrics as diagnostic tools rather than targets. Includes the family love framing for qualitative data. |

## Contributing

To add an analogy, create a file in `analogies/` and add it to the table above. A good analogy file includes:

- The core concept in plain language
- How it maps to the DevEx problem
- Scripts for different audiences (technical teams, executives, skeptics)
- Common objections and how to handle them
- When to use it and when not to

If you're adding a new analogy that's still being tested, note that in the file so others know not to rely on it too heavily yet.
